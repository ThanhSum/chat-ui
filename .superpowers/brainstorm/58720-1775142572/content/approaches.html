How should we approach this?

Three ways to deliver model UX + input bar + message polish in GitHub style

1

Targeted component upgrades ★ Recommended

Improve each of the 3 areas independently, keeping the existing layout shell intact. Ollama status dot, richer model card, slash commands in the input bar, better code blocks — surgical changes only.

Pros

  • Lowest risk
  • Easy to review piece by piece
  • No layout regressions

Cons

  • Less visually unified
  • Each component styled slightly independently
2

GitHub design tokens + features

Define CSS variables mirroring GitHub's dark palette (--color-border-default, --color-fg-muted, etc.) and apply them across all components in one pass, while adding all 3 feature sets.

Pros

  • Most visually cohesive
  • Easier to maintain long-term

Cons

  • More touchpoints = higher risk
  • Requires careful Tailwind integration
3

Full feature expansion

Everything in option 1, plus structural additions: collapsible sidebar, ⌘K command palette (search conversations + switch models), and a sticky model/context bar in the chat header.

Pros

  • Most impactful UX
  • Keyboard-driven power-user flow

Cons

  • Touches app shell layout
  • More implementation work
